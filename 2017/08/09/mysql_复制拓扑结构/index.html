mysql 主从拓扑结构

MySQL的主从拓扑结构在少量写大量读的情况下表现非常优越。可以把读分摊到多个备库上，直到写成为整个拓扑结构的瓶颈(在容量规划会介绍为什么写会成为瓶颈)。这种拓扑结构在其他拓扑结构中，是比较简单的一种，但很灵活，能满足多种要求。

把一台备库当做待用的主库，除了复制没有其他的数据传输。
把一台备库放在远程数据中心，用作灾难恢复
延迟一个或多个备库，以备灾难恢复
使用一个备库作为开发测试等
